# PAD-13: Governance (CoC, Contributing, Maintainers)

**Project:** MAP Protocol Open Source Release  
**Status:** Complete  
**Last Updated:** 2026-03-30

## 1. Code of Conduct

MAP Protocol follows the **Contributor Covenant Code of Conduct**.

### 1.1 Our Pledge

We pledge to make participation in our project a harassment-free experience for everyone.

### 1.2 Our Standards

**Acceptable behavior:**
- Using welcoming and inclusive language
- Being respectful of differing viewpoints
- Gracefully accepting constructive criticism
- Focusing on what is best for the community

**Unacceptable behavior:**
- Trolling, insulting/derogatory comments
- Personal attacks or public harassment
- Publishing others' private information
- Other conduct deemed inappropriate

### 1.3 Enforcement

Instances of abusive, harassing, or otherwise unacceptable behavior may be reported to the maintainers. All complaints will be reviewed and investigated.

### 1.4 File

- `CODE_OF_CONDUCT.md` - Full Code of Conduct

## 2. Contributing Guidelines

### 2.1 Getting Started

1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Add tests
5. Submit a PR

### 2.2 Development Setup

```bash
git clone https://github.com/mapprotocol/map
cd map
npm install
```

### 2.3 Code Standards

- Follow existing code style
- Add tests for new features
- Update documentation
- Use conventional commits

### 2.4 Pull Request Process

1. PR description must explain the change
2. All CI checks must pass
3. At least one maintainer review required
4. Changes must be approved before merge

### 2.5 File

- `CONTRIBUTING.md` - Full contributing guide

## 3. Maintainers

### 3.1 Governance Model

MAP Protocol uses a **benevolent dictator** model with maintainer oversight.

**Maintainers:**
- Review and approve changes
- Enforce code quality standards
- Resolve disputes
- Set project direction

### 3.2 Responsibilities

- Review PRs within 48 hours
- Maintain code quality
- Update documentation
- Release management

### 3.3 File

- `MAINTAINERS.md` - Governance document

## 4. Issue Templates

### 4.1 Bug Report

Template for reporting bugs with:
- Description
- Steps to reproduce
- Expected vs actual behavior
- Environment details

### 4.2 Feature Request

Template for requesting features with:
- Problem statement
- Proposed solution
- Alternatives considered
- Additional context

### 4.3 Security Issue

Template for security vulnerabilities with:
- Vulnerability description
- Impact assessment
- Proof of concept
- Suggested fix
